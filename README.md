# Web Worker

Небольшой демо-проект на чистом JavaScript, который показывает базовую работу с `Web Worker`.

Приложение принимает число из формы, отправляет его в отдельный поток и вычисляет число Фибоначчи вне основного UI-потока. Это простой пример того, как вынести тяжелую синхронную задачу из main thread, чтобы интерфейс не зависал.

## Что здесь происходит

- `index.html` содержит простую форму для ввода числа и вывода результата
- `main.js` работает в основном потоке, создает `Worker` и обменивается сообщениями
- `worker.js` принимает число, считает `fibonacci(n)` и отправляет результат обратно

## Как работает обмен сообщениями

1. Пользователь вводит число и отправляет форму.
2. `main.js` создает `new Worker('./worker.js')`.
3. Основной поток отправляет число через `postMessage`.
4. `worker.js` вычисляет результат и возвращает его через `postMessage`.
5. Основной поток получает ответ в `onmessage` и показывает результат в форме.

## Зачем здесь Web Worker

Рекурсивный расчет Фибоначчи является CPU-bound задачей. Если считать его прямо в основном потоке, интерфейс может подвисать. `Web Worker` позволяет выполнить вычисление отдельно и не блокировать работу страницы.

## Ограничения примера

- используется наивный рекурсивный алгоритм `fibonacci`, который быстро становится тяжелым на больших числах
- при каждом расчете создается новый worker
- нет валидации ввода и обработки некорректных значений

## Зачем этот репозиторий

Проект подходит как короткий учебный пример для портфолио:

- базовое использование `Web Worker`
- обмен сообщениями между потоками
- отделение тяжелых вычислений от UI
